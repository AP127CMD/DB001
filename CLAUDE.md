## AP127 Detail Sync System

`index.html` and `student.html` (and therefore `AP127_DashboardR1/index.html`) are kept identical in their AP127 Detail sections via comment markers and two scripts.

### Markers (present in both `index.html` and `student.html`)

| Marker pair | What it covers |
|---|---|
| `/* ##AP127CSS_START## */` … `/* ##AP127CSS_END## */` | All `.d127-*` CSS classes |
| `<!-- ##AP127PAGE_START## -->` … `<!-- ##AP127PAGE_END## -->` | Entire `#page-ap127detail` HTML |
| `<!-- ##AP127DRAWER_START## -->` … `<!-- ##AP127DRAWER_END## -->` | Toast + drawer overlay HTML |
| `// ##AP127NICKS_START##` … `// ##AP127NICKS_END##` | `AP127_NICKS`, `AP127_FI`, `AP127_SE` and `HOL` constants |
| `// ##AP127JS_START##` … `// ##AP127JS_END##` | All `ap127*` helper + render + chart functions |

### Workflow (automatic on every push to `main`)

```
1. node update-cache.js       → rebuilds cache.json
2. node build-student.js      → patches student.html from index.html markers
3. git add cache.json student.html && git commit && git push
4. node push-to-kv.js         → pushes AP127 slice to Cloudflare KV
5. node sync-dashboardr1.js   → pushes student.html (WORKER_URL injected) to AP127_DashboardR1
6. sed inject secrets          → for GitHub Pages artifact only (main only)
7. upload Pages artifact + deploy
```

### Developer workflow

When making changes to the AP127 Detail view:
- **Only edit `index.html`** (the marked sections)
- Commit and push to `main` → GitHub Actions runs `build-student.js` automatically
- Both `student.html` in this repo and `AP127_DashboardR1/index.html` are updated automatically
- **Do not manually edit the marked sections in `student.html`** — they will be overwritten on next run

Sections outside the markers in `student.html` (nav, `WORKER_URL`, global vars, init IIFE) are student-specific and are never overwritten.

## Cloudflare Infrastructure

| Resource | Name / URL | Notes |
|---|---|---|
| KV Namespace | `AP127_STUDENT_DATA` | Stores key `ap127_slice` |
| Worker | `ap127-data-api` | Serves `ap127_slice` as JSON. `ALLOWED_ORIGIN` = `https://ap127-dashboardr1.pages.dev` |
| Pages project | `ap127-dashboardr1` | Deploys from `nuguitar/AP127_DashboardR1` (private) on push to `main` |
| Student site URL | `https://ap127-dashboardr1.pages.dev` | Live and serving data |

### Cloudflare Worker (`ap127-data-api`) code

```javascript
export default {
  async fetch(request, env) {
    const allowedOrigin = env.ALLOWED_ORIGIN || '*';
    if (request.method === 'OPTIONS') {
      return new Response(null, { headers: {
        'Access-Control-Allow-Origin': allowedOrigin,
        'Access-Control-Allow-Methods': 'GET',
        'Access-Control-Max-Age': '86400'
      }});
    }
    if (request.method !== 'GET') return new Response('Method not allowed', { status: 405 });
    const data = await env.KV.get('ap127_slice', 'json');
    if (!data) return new Response(JSON.stringify({ error: 'No data' }), {
      status: 503,
      headers: { 'Content-Type': 'application/json' }
    });
    return new Response(JSON.stringify(data), { headers: {
      'Content-Type': 'application/json',
      'Cache-Control': 'no-store',
      'Access-Control-Allow-Origin': allowedOrigin
    }});
  },
};
```

KV binding: variable name `KV` → namespace `AP127_STUDENT_DATA`

3. Permissions: **Contents → Read and Write**
4. Add as secret `GH_PAT_DASHBOARDR1` to `AP127_NGT_001`

## GitHub Actions Workflow (`update-cache.yml`) — Key Points

- Split into two jobs to avoid GitHub Pages environment protection errors on non-main branches:
  - `update` job: runs on **any branch** (no environment). Steps: fetch data → sync student.html → commit cache.json + student.html → push KV → push to DashboardR1 (main only) → inject secrets (main only) → upload Pages artifact (main only)
  - `deploy` job: runs on **main only**, has `environment: github-pages`. Only calls `actions/deploy-pages@v5`
- Steps with `if: github.ref == 'refs/heads/main'` guard: DashboardR1 sync, secret injection, setup-pages, upload-artifact

## Student Site (`AP127_DashboardR1`) — How It Works

- `index.html` is auto-synced from `student.html` in this repo by `sync-dashboardr1.js` on every push to `main`
- `WORKER_URL` is replaced with `CF_WORKER_URL` secret value during sync (never stored as plain text in either public file)
- Init IIFE: fetches from `WORKER_URL` if set, else falls back to `cache.json` locally (local dev only)
- No relay URL, no admin hash, no sync button — student view only
